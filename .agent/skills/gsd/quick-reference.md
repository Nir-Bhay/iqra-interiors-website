# GSD - Quick Reference Guide

A quick cheat sheet for using the **Get Started Done** skill.

## 🚀 Quick Start

**Basic Usage:**
```
gsd [task description]
```

**Examples:**
```
"gsd add a contact form to my website"
"gsd fix the mobile layout on homepage"
"gsd optimize image loading speed"
"gsd add dark mode toggle"
```

## 📋 GSD Phases

### 1. GET - Define
- What needs to be done?
- What does "done" look like?
- What's the scope?

### 2. START - Execute
- Implement the solution
- Follow existing patterns
- Focus on minimal viable solution

### 3. DONE - Verify
- Test functionality
- Check for errors
- Confirm completion

## ✅ When to Use GSD

| Do Use GSD For | Don't Use GSD For |
|---------------|-------------------|
| Add a contact form | Build entire e-commerce site |
| Fix broken button | Analyze entire codebase |
| Add dark mode | Choose architecture |
| Optimize images | Plan multi-feature project |
| Fix mobile layout | Make major framework decisions |

## 🎯 GSD vs Other Skills

| Your Task | Use |
|-----------|-----|
| Quick fix or feature | **GSD** |
| New complex project | `app-builder` |
| Multi-step planning | `project-planner` |
| Codebase analysis | `explorer-agent` |
| Complex UI/UX | `frontend-specialist` |

## 📝 What to Expect

When you use GSD, you'll see:

```
🎯 GET - Task Definition
**Task:** [Your task]
**Scope:** [What's included]
**Done means:** [Acceptance criteria]

🚀 START - Execution
[Implementation details]

✅ DONE - Completion
**What was done:** [Summary]
**How to verify:** [Testing instructions]
**Next steps:** [Optional improvements]
```

## 🕐 Typical Timeframes

| Task Type | Typical Duration |
|-----------|------------------|
| Small UI fix | 15-30 minutes |
| Add simple feature | 30-60 minutes |
| Debug issue | 20-45 minutes |
| Optimization | 30-90 minutes |

## 🔍 Verification

Always test the result:

1. **Visual:** Check it looks right
2. **Functional:** Test it works as expected
3. **Error-free:** No console errors
4. **Cross-device:** Test on mobile/desktop if relevant

## 💡 Pro Tips

- **Be specific:** "Add contact form with name, email, message" > "Add form"
- **Define scope:** "Just the form, no backend integration yet"
- **Test incrementally:** Try to verify as we go when possible
- **Ask questions:** If you're unsure what you want, that's okay!

## 🚦 GSD Command Checklist

Before using GSD:
- [ ] Is it a single, focused task?
- [ ] Can it be completed in under 2 hours?
- [ ] Do you have a clear idea of what you want?

## 🎯 Success Indicators

GSD was successful when:
- ✅ Task is complete as requested
- ✅ No breaking changes to existing code
- ✅ Code follows your project patterns
- ✅ You can verify the result easily

---

**Quick Questions to Ask Yourself:**

- *What exactly do I need?*
- *Where should it go?*
- *What does "done" mean to me?*

If you can answer these, GSD is a great choice!
